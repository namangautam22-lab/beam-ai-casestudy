Section 3 — Solution Concept

A three-layer review system

The insight: not all 400 annotations deserve equal attention. The solution isn't a faster way to check everything — it's a smarter way to know what to skip.

Layer 1
Confidence Engine
AI scores every annotation 0–100 on detection confidence.
Inputs:
Symbol clarity Overlap detection Drawing scale Historical accuracy
→ ↓
Layer 2
Risk Prioritizer
Combines confidence + dollar value + annotation rarity. Outputs a ranked review queue.

Key assumptions — stated explicitly
1

Confidence scores already exist. Beam AI's model produces a per-annotation confidence score internally. Quick Review surfaces this — it does not require training a new model. This is a UI change, not an ML change.

2

Dollar values are seeded, not custom. Item values come from a standard HVAC price reference (RSMeans or equivalent) seeded into the platform. Estimators can override per project. No custom pricing engine required for v1.

3

"Review complete" is not 100%. It means all High-Risk items reviewed plus overall review exceeding 60% complete. This is a deliberate design choice — perfect coverage is the enemy of shipping bids.

⚖
Key trade-off: false positives are acceptable

The prioritization model will sometimes surface false positives — items the AI was uncertain about that turn out to be correct. This is acceptable. The cost of an unnecessary check is 10 seconds. The cost of a missed misidentified AHU is $40,000. We bias toward over-flagging, not under-flagging.

Section 5 — Rollout Plan

How to ship this in three moves

Each version builds evidence for the next. We don't ship the full prioritization logic until we've validated that surfacing uncertainty changes estimator behavior.

v0 Weeks 1–4
"Confidence Surfacing"
Zero model changes required
Scope

Show AI confidence score on every annotation. No queue ranking yet. Just make uncertainty visible to the estimator.

Metric to watch

Do estimators spend more time on low-confidence items? Measured via time-on-item telemetry per annotation.

Why this first

Zero new model work. Uses existing confidence scores already produced by Beam's pipeline. Proves the hypothesis — that surfacing uncertainty changes review behavior — before building the full ranking system.

v1 Weeks 5–12
"Risk-Ranked Queue"
Core product — ship when v0 validates
Scope

Add dollar-value weighting, ranked queue, keyboard shortcuts [A]/[C]/[F], and progress bar showing % reviewed vs. review complete threshold.

Metric to watch

Review completion time per drawing. Target: under 25 min for a 40-sheet job.

Why this order

v0 data tells us which item categories estimators over-check. v1 uses that to calibrate the risk model before shipping it to all users. We're not guessing at the weights — we're learning them.

v2 Weeks 13–20
"Smart Review Complete"
Export gate + session summary
Scope

"Review Complete" state triggered when all High-Risk items are reviewed and overall review exceeds 60%. Export is gated on this state. A session summary appears before export showing what was accepted, corrected, and flagged.

Metric to watch

Post-export error rate vs. pre-Quick Review baseline.

Why export gate

The gate is the product making a commitment to the estimator: "this is complete enough to bid on." Without it, "Review Complete" is advisory and gets ignored.

⚡
The risk to test before full build

The prioritization model assumes AI confidence scores are calibrated — that a 61% confidence score genuinely reflects a higher error rate than a 91% score, at scale, across drawing types and carriers.

If confidence scores are not calibrated — if the AI is equally uncertain about everything — the entire prioritization layer produces no value. High-risk items surface, but they're no more likely to be wrong than low-risk ones. The estimator learns this within a week and stops trusting the queue.

The test to run before v1 ships

Take 50 historical drawings with known correction data. Rank every annotation by AI confidence score. Verify that items in the bottom confidence quartile were corrected at a meaningfully higher rate than the top quartile. If the correction rate doesn't differ by at least 2×, we redefine "risk" using item dollar value alone — and note that the confidence score is decorative, not predictive. Either outcome is useful. Not testing this before shipping v1 is not.
