Solution Architecture

Three-layer architecture

Each layer builds on the last. The key constraint: no new ML models. Everything surfaces what Beam already computes.

🎯
Confidence Engine
Scores each annotation 0–100 based on symbol clarity, bounding box overlap, drawing scale consistency, and historical accuracy for that item type.
→
⚡
Risk Prioritizer
Combines confidence score, estimated dollar value, and item rarity into a ranked queue. Low confidence + high value = top of queue.
→
✓
Quick Review UX
Keyboard-first interface surfaces drawing context inline. Progress bar and export gating prevent premature BoQ submission.
Key Trade-off

False positives are acceptable. Cost of an unnecessary check: ~10 seconds. Cost of a missed AHU miscount: $40,000+. The system deliberately biases toward over-flagging.

Assumptions — stated explicitly

  1. Confidence scores already exist in Beam's ML pipeline. This is a UI surface change, not a new model.
  2. Dollar values come from RSMeans 2024 HVAC reference data, not live quote feeds.
  3. "Review complete" ≠ 100% coverage. Threshold: all High-Risk items cleared + 60% overall.

Rollout Plan

Three-phase strategy

Each phase ships something estimators can use immediately, then instruments the data needed to justify the next phase.

Phase 1Weeks 1–3
Signal Only
Add confidence score badge to existing annotation list. No reordering. No new UI chrome. Ships as a feature flag to 2 beta estimators.
Exit criterion
Estimators can describe, unprompted, which items are high-confidence vs low-confidence. Validated by session recording.
Phase 2Weeks 4–10
Ranked Queue + Review Tracking
Risk-prioritized list, keyboard shortcuts, progress bar, inline detail view. Export remains ungated — we observe behaviour before enforcing.
Exit criterion
Median review time under 40 min across 10 sessions. High-risk items opened before medium items in 80%+ of sessions.
Phase 3Weeks 11–18
Export Gating + Post-Export Instrumentation
Block BoQ export until all high-risk items actioned. Session summary on export. Flagged items routed to project lead queue. Field error tracking (requires Procore or equivalent API — dependency to confirm Week 6).
Exit criterion
Post-export error rate drops below 1.5% across 20+ export events.
Biggest risk: Phase 3 field integration

The biggest risk is Phase 3's field integration dependency. If Procore access isn't confirmed by Week 6, the post-export error metric can't be instrumented. Mitigation: manual field correction log as a fallback (adds 2 weeks).
